# Инструкция по добавлению фотографий в мобильную версию

## Структура директорий

Для корректной работы мобильного приложения необходимо создать следующую структуру:

```
/images/
├── routes/      // Фотографии маршрутов (800x400px)
└── points/      // Фотографии точек маршрутов (600x400px)
```

## Оптимизация для мобильных устройств

1. **Форматы изображений:**
   - Используйте формат WebP для лучшего сжатия
   - Fallback в формате JPG для старых устройств

2. **Размеры файлов:**
   - Маршруты: не более 150 КБ
   - Точки: не более 100 КБ

3. **Адаптивность:**
   - Используйте атрибут `srcset` для разных экранов
   - Добавьте lazy loading для экономии трафика

## Пример кода для адаптивных изображений

```html
<img 
  src="images/routes/1.jpg" 
  srcset="images/routes/1-small.jpg 480w, images/routes/1.jpg 800w" 
  sizes="(max-width: 480px) 480px, 800px"
  loading="lazy"
  alt="Название маршрута">
```

## Именование файлов

- Фотографии маршрутов: `[id-маршрута].webp` (например, `1.webp`)
- Фотографии точек: `[id-маршрута]-[номер-точки].webp` (например, `1-1.webp`)

## Обновление JSON

В файле `data/routes_with_photos.json` укажите пути к изображениям:

```json
{
  "id": 1,
  "title": "Название маршрута",
  "image": "1.webp",
  "points": [
    {
      "title": "Точка 1",
      "image": "1-1.webp"
    }
  ]
}
```

## Советы по фотографиям

1. **Маршруты:**
   - Используйте пейзажную ориентацию
   - Покажите характерные места маршрута
   - Добавьте человека для масштаба

2. **Точки:**
   - Фокусируйтесь на деталях
   - Покажите атмосферу места
   - Используйте разное время суток для разнообразия

## Генерация WebP

Для конвертации изображений используйте:
- Онлайн-сервисы: Squoosh, TinyPNG
- Инструменты: cwebp, imagemin
- WordPress: плагины оптимизации изображений
